HoopFire

HoopFire is a sports prediction web application. Two friends and I came up with the idea a few years back and it has been a pet project ever since. It was originally built using Vaadin and was rewritten in 2016. The front end is now built using AngularJS with Angular Material design principles. The data is persisted using Hibernate against a MySQL database with application dependencies being managed by Spring. The application build is handled by Maven and we're using Heroku to host the application.

Comic Info Search

I came across the Comic Vine API and thought it was something very interesting. I decided I wanted to try and create some kind of website using the API. I settled on a search website. The user can search for any character they wish and the will return a list of results. From there, the user can choose a character and the site will display a photo and all the information about that character. I chose a 2 sided design to showcase the character images while still displaying all the information. The overall layout is designed to feel like the user is looking at a comic book page, with the gaps and border mimicing that of the comic style. The background colors are dynamic and change with the user selected character by taking advantage of VibrantJS. The site is built using NPM, Bower, Grunt, and AngularJS. I use a Github repository thats hooked up to Netlify to allow auto-publishing for any change I commit. Click the link to check it out!

Plex Request

Plex Request started as a joke between my friends and I. We wanted a way to browse movies and Tv shows we may want to watch later. It simply sends a user selected list to a friend of mine who gracioulsy volunteered to maintain the full list. All information is a product of the JSON returned from using the The MovieDB API. The original site was built using Yeoman to set up the scaffolding, which includes NPM, Bower, and Grunt. The site is implemented with AngularJS and Angular Material Design. That site can be found here. I've since redone the site. It's still uses AngularJS and Angular Material but it now manged by Gulp and is a more visually appealing.
